```python
"""
RBAC ORM Models — Role and UserRole (Phase 3)
==============================================
Tables:
  - roles       — 3 built-in roles (org_admin, workspace_member, project_viewer)
  - user_roles  — junction: user + role + scoped resource
"""
import uuid
from datetime import datetime, timezone
from sqlalchemy import DateTime, ForeignKey, String, Index
from sqlalchemy.orm import Mapped, mapped_column
from app.db.models import Base

# Static role IDs for built-in roles (deterministic, like DEFAULT_ORG_ID pattern)
ROLE_ORG_ADMIN_ID = "00000000-0000-0000-0000-role00000001"
ROLE_WORKSPACE_MEMBER_ID = "00000000-0000-0000-0000-role00000002"
ROLE_PROJECT_VIEWER_ID = "00000000-0000-0000-0000-role00000003"

class Role(Base):
    __tablename__ = "roles"
    id: Mapped[str] = mapped_column(String, primary_key=True, default=lambda: str(uuid.uuid4()))
    name: Mapped[str] = mapped_column(String, nullable=False, unique=True)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), default=lambda: datetime.now(timezone.utc)
    )

class UserRole(Base):
    __tablename__ = "user_roles"
    id: Mapped[str] = mapped_column(String, primary_key=True, default=lambda: str(uuid.uuid4()))
    user_id: Mapped[str] = mapped_column(
        String, ForeignKey("users.id", ondelete="CASCADE"), nullable=False
    )
    role_id: Mapped[str] = mapped_column(
        String, ForeignKey("roles.id", ondelete="CASCADE"), nullable=False
    )
    resource_type: Mapped[str] = mapped_column(String, nullable=False)  # "organization"|"workspace"|"project"
    resource_id: Mapped[str] = mapped_column(String, nullable=False)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), default=lambda: datetime.now(timezone.utc)
    )
    __table_args__ = (
        Index("ix_user_roles_lookup", "user_id", "resource_type", "resource_id"),
    )
```

**`can_user(user_id, action, resource_type, resource_id, db, request)` -> bool:**
- If `not settings.ENFORCE_AUTH`: return True immediately (per D-09)
- Check `request.state.rbac_cache` dict (initialize as empty dict on first call via `hasattr` check) — if cache hit, return cached value (per D-13)
- Check if user is superadmin: `user = await db.get(User, user_id)` — if `user.is_superadmin`: cache True, return True
- Call `_resolve(user_id, action, resource_type, resource_id, db)` for the actual hierarchy traversal
- Cache and return the result

**`_resolve(user_id, action, resource_type, resource_id, db)` -> bool:**
- Call `_has_role_with_action(user_id, action, resource_type, resource_id, db)` — if True, return True (direct role match)
- If `resource_type == "project"`: fetch `Project` by `resource_id`; if not found return False (IDOR — unknown project is denied); if `project.workspace_id` is set, check workspace role; then check org role via `project.organization_id`
- If `resource_type == "workspace"`: fetch `Workspace` by `resource_id`; if found, check org role via `workspace.organization_id`
- Return False

**`_has_role_with_action(user_id, action, resource_type, resource_id, db)` -> bool:**
- `SELECT user_roles JOIN roles ON user_roles.role_id = roles.id WHERE user_roles.user_id = :user_id AND user_roles.resource_type = :resource_type AND user_roles.resource_id = :resource_id`
- For each matching row, check if `action in ROLE_PERMISSIONS.get(role.name, set())`
- Return True on first match, False if none match

**`require_permission(action, resource_type)` -> Depends callable:**
- Returns an async inner function `_check(request, project_id=None, current_user=Depends(get_current_user), db=Depends(get_db)) -> None`
- If `not settings.ENFORCE_AUTH`: return None (per D-09)
- If `isinstance(current_user, AnonymousUser)`: return None (handles edge case)
- Determine `resource_id`: if `resource_type == "project"` use `project_id` path param; if `resource_type == "organization"` use the project's org_id or `DEFAULT_ORG_ID` (per D-06 — for list/create routes without project_id)
- Call `can_user(...)` — if False, raise `HTTPException(403, detail="Brak uprawnien do tego zasobu.")` (per UI-SPEC)

Important: the `project_id` parameter in `_check` must have a default of `None` so routes without `{project_id}` path param don't get a 422. For org-scoped routes, `resource_id` falls back to looking up the user's org or using `DEFAULT_ORG_ID`.
